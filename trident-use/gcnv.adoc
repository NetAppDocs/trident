= Configure Google Cloud NetApp Volumes for NAS workloads

:imagesdir: ../media/

[.lead]
You can configure Trident to provision NAS storage volumes using the NFS or SMB
protocols from Google Cloud NetApp Volumes. NAS volumes are provisioned by using
the `google-cloud-netapp-volumes` storage driver.

You can deploy shared, file-based workloads on Google Kubernetes Engine using
Trident-managed NAS storage from Google Cloud NetApp Volumes.

This driver is dedicated to file-based workloads and does not support block
or iSCSI volumes.

== NAS volumes and iSCSI block volumes

Google Cloud NetApp Volumes supports both NAS and block storage, which differ in
how applications access and manage data.

NAS volumes provide file-based storage and are accessed through standard file
protocols such as NFS or SMB. Volumes are mounted as shared filesystems and support
concurrent access from multiple pods or nodes. NAS storage is well suited to
workloads that require shared read-write access, filesystem semantics, and minimal
node-level configuration.

iSCSI block volumes provide raw block storage and are accessed as block devices
attached to Kubernetes nodes. Block storage is typically used when applications
require block-level access or tighter control over I/O behavior.

This applies to the following environments:

* Trident 26.02 and later
* Google Kubernetes Engine (GKE)
* Google Cloud NetApp Volumes NAS pools
* NFS and SMB workloads

For block (iSCSI) workloads, see
xref:trident-use/gcnv-block.html[Configure block storage (iSCSI)].

== NAS storage model

NAS storage provides shared, file-based access to data. Volumes are mounted as
filesystems and accessed using standard POSIX or SMB semantics.

Trident manages the full lifecycle of NAS volumes, including volume creation,
export configuration, mounting, and deletion.

== When to use NAS storage

Use NAS storage when your workload requires:

* Shared read-write access from multiple pods or nodes
* Filesystem semantics
* Minimal node-level storage configuration
* Support for the `ReadWriteMany` access mode

== NAS volumes

NAS volumes are provisioned as file volumes in Google Cloud NetApp Volumes and
mounted into pods as directories.

NAS volumes:

* Use the NFS or SMB protocols
* Are mounted as filesystems
* Support multi-node access
* Do not require iSCSI or multipath configuration

== Supported protocols

The NAS driver supports the following protocols:

* NFS
* SMB

== Access modes

NAS volumes support the following Kubernetes access modes:

* `ReadWriteOnce` (RWO)
* `ReadOnlyMany` (ROX)
* `ReadWriteMany` (RWX)
* `ReadWriteOncePod` (RWOP)

== volumeMode behavior

NAS volumes always use `volumeMode: Filesystem`.

== Storage driver

NAS workloads use the following Trident storage driver:

[cols="2,2,2,2", options="header"]
|===
|Driver |Protocols |volumeMode |Usage

|`google-cloud-netapp-volumes`
|NFS, SMB
|Filesystem
|NAS workloads
|===

== Configure a Trident NAS backend

[source,yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentBackendConfig
metadata:
  name: gcnv-nas
  namespace: trident
spec:
  version: 1
  storageDriverName: google-cloud-netapp-volumes
  projectNumber: "<project-number>"
  location: "<region>"
  sdkTimeout: "600"
  storage:
  - labels:
      cloud: gcp
    network: "<vpc-network>"
----
